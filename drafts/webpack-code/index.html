Как устроен Webpack: смотрим исходники

Webpack - это, стремительно набирающий популярность, сборщик модулей. Он прост в обращении, предоставляет большие возможности для организации кода, объединяет уже существующие подходы модулирования приложений и дополняет их. В статье мы не будем аргументировать зачем и когда использовать webpack, подразумевается, что читатель уже знаком с проектом и надобности мотивировать его нет. Мы же попытаемся заглянуть внутрь и понять как это работает с помощью разбора примеров из документации.

Getting Started

Проект оформлен в виде организации на гитхабе:

  • основной проект https://github.com/webpack/webpack
  • загрузчики - например https://github.com/webpack/jade-loader
  • плагины - например https://github.com/webpack/component-webpack-plugin
  • лендинг https://github.com/webpack/webpack.github.com
  • документация https://github.com/webpack/docs
